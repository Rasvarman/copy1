### Задание 1

1.1 - Подойдёт полное резервное копирование (Full Backup)  выполняемое ежедневно.
Простота восстановления, нужен только последний backup.
Минимизация риска потери данных.
Если мало ресурсов , то Дифференциальное копирование - сохраняет все изменения с момента последнего полного backup,  меньше файлов для восстановления, чем в инкрементном.

1.2 - Инкрементное – копируются только изменения с последнего backup + WAL - логи. 
Экономия места. Точное восстановление - WAL логи способны откатить систему на любой момент.

1.3 - Master-Slave репликация – при падении мастера можно переключиться на slave.

### Задание 2
Резервное копирование 

pg_dump -U username -d dbname -F c -f backup.dump

Восстановление 
pg_restore -U username -d dbname -C backup.dump
